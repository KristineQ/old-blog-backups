主要是关于 createElement API 的原理，JSX 的简单语法，props，事件调用，以及 style 的理解所整理的笔记

createElement API

React 定义了自己的函数 createElement ，它的作用和 document.createElement很相似。React 也使用 ReactDOM里的render函数，来替代 appendChild。就是相当于把正常的添加一个节点的过程简化，包装到一个函数里面那个样子。

JSX

JSX 其实就是可以代替上面那个函数的一种简写的方式，本质上还是 create 了一个元素。

不能直接在只引入 React 的情况下用，还需要把 JSX 转换成 createElement 函数才能使用。使用 babel 可以实现这种转换。

在 JSX 中可以用花括号添加任何 JavaScript 代码，也可以是 JSX 表达式。

使用花括号可以更好的传递参数，也可以按照 createElement 那样把想要传递到第二个参数的那些值放到一个对象里，然后用...运算符来解析,用组件的特有的值替代 props 里面的值，只需要在解析后面重写覆盖就可以了。

花括号只能解析 js 表达式，所以如果用 if 之类的代码块就不能解析出来，对于只有是否判断的 JSX 可以使用三元运算符，但是如果判断的情况比较多可以考虑写一个组件。

React Component

在 JSX 中的{}里面写函数，就会正常的在页面渲染添加一个标签。

这个函数是 JSX 事实上就也是使用 createElement API 渲染的函数，用 babel 转换可以看出来实际的函数情况

React 的 createElement 函数第一个变量可以是一个字符串类似‘div’，也可以是一个函数，这样的话会返回按照函数里面的 JSX 渲染的值所在的渲染，逐层往里渲染，直到最后一个是字符串渲染为止。

但是如果当作一个组件直接类似这样写的话，渲染就会报错

事实上，以上按照 babel 的转换规则，就会是一个’message’，类似的字符串变量作为 createElement 第一个参数，然后就会报错说找不到这样的浏览器标签。而如果按照 React 规则，把第一个字母用大写来的话，则会当作函数作为 createElement 第一个参数，来执行渲染函数。

这样的规则的原因有可能是因为浏览器本身就没有以大写字母开头的标签，如果出现大写字母开头的标签，就会当作是 React 自定义标签来解析。

此时我们还需要把原来作为函数的变量也首字母大写

propTypes

判断自定义 prop 的类型。需要引入一个 propTypes 的包，然后在 class 中构建一个静态变量

在生产环境中会没有 propTypes 的包，使用 npm 加入 babel-plugin-transform-react-remove-prop-types 这个包，在你为生产构建它时自动从你的代码中删除 PropTypes

Render

React 的render()函数是非高效的，它只会重复渲染需要更新的部分，类比普通的更新整个 innerHtml 的 js 方法，React 不会重新渲染整个 div。

Style

可以用 style 来给 box 添加样式，也可以用 className，在组件内部写好默认的样式，然后在参数中写特殊的样式之类的

这里浅拷贝相当于重写了 style 这个对象，然后我们可以用...运算符来重写一下 Box 这个组件，还可以增加自定义样式 size

这样就可以实现既有基础样式又可以往上面增加新的样式辽

Event Handlers

如果想要给这个标签组件添加一个自定义事件函数，可以直接在属性中加入箭头函数

或者可以用一个新的函数封装

如果是一个输入事件，也可以使用 event 传递，用event.target.value获取输入值。

onblur 事件会在对象失去焦点时发生。

React 在合成了一个事件系统。使用 event.nativeEvent 可以访问原始事件。保存就可以得到了真正的原生事件。好处是 React 正在为我们优化事物，因此它使用事件代理。对于整个文档，每种类型实际上只有一个事件处理程序处理事件。

Class Component

使用 Class 形式的组件的时候，要在组件调用的方法后边.bind(this)这是类似原生 js 的一些常见错误,那么为什么要在调用的时候 bind 上这个 this 呢

就相当于原生 JavaScript 中，调用函数的作用域和原函数的作用域不同导致的找不到相应变量的错误相似

另外 constructor 中的 this 指向的是原型的环境，所以我们也可以在 constructor 中重新 bind，使得 handleClick 的作用环境是当前 class 的环境。实际上是在 constructor 中利用this.handleClick创建了新函数，然后用 bind 了当前 class 环境的 handleClick 函数传递给这个新函数，调用的时候是直接在原型上查找 handleClick，调用环境还是当前 class。所以可以在没有.bind(this)的情况下传递它

其实我们可以在没有太多参数需要传给 constructor 的情况下直接把 state 提到公共类,然后 bind 方法到当前环境，也是可以直接利用箭头函数写在当前环境

所以最终是以这种形式——Public Class Fields syntax（实例可以访问类字段的语法）来构造需要的组件

ref

如果想要引用一个用 jQuery 或者是原生 js 写的组件到 react 里，而且需要访问原本的 DOM 结构，这个时候就可以使用ref来实现
