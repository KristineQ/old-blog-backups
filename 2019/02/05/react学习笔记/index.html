createElement API

React定义了自己的函数

JSX

JSX其实就是可以代替上面那个函数的一种简写的方式，本质上还是create了一个元素。

不能直接在只引入React的情况下用，还需要把JSX转换成createElement函数才能使用。使用babel可以实现这种转换。

在JSX中可以用花括号添加任何JavaScript代码，也可以是JSX表达式。

使用花括号可以更好的传递参数，也可以按照createElement那样把想要传递到第二个参数的那些值放到一个对象里，然后用

花括号只能解析js表达式，所以如果用if之类的代码块就不能解析出来，对于只有是否判断的JSX可以使用三元运算符，但是如果判断的情况比较多可以考虑写一个组件。

React Component

这个函数是JSX事实上就也是使用createElement API渲染的函数，用babel转换可以看出来实际的函数情况

React的createElement函数第一个变量可以是一个字符串类似‘div’，也可以是一个函数，这样的话会返回按照函数里面的JSX渲染的值所在的渲染，逐层往里渲染，直到最后一个是字符串渲染为止。

但是如果当作一个组件直接类似这样写的话，渲染就会报错

事实上，以上按照babel的转换规则，就会是一个’message’，类似的字符串变量作为createElement第一个参数，然后就会报错说找不到这样的浏览器标签。而如果按照React规则，把第一个字母用大写来的话，则会当作函数作为createElement第一个参数，来执行渲染函数。

这样的规则的原因有可能是因为浏览器本身就没有以大写字母开头的标签，如果出现大写字母开头的标签，就会当作是React自定义标签来解析。

此时我们还需要把原来作为函数的变量也首字母大写
