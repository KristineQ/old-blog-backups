Node.js is a JavaScript runtime built on Chrome’s V8 JavaScript engine. –(source)

  • 所有 JavaScript 代码均由 V8 在同一线程中运行。
  • JavaScript 回调由事件循环调用，并由 V8 引擎运行。
  • 事件循环是启用 Node 的非阻塞 I / O 模型的原因，它由多个阶段组成，在这些阶段中，将调用回调（如果有要调用的回调）,使用事件循环可以解决单线程的 I/O 的问题（这里有个特别清楚的文章讲解执行栈同步异步执行过程）
  • 阻塞 I / O – V8 线程必须等待 I / O 操作完成，并在此期间被阻止，直到结果可用为止。阻止 I / O 是同步的，因为它阻止 V8 引擎运行其他 JavaScript 代码。
  • 非阻塞 I / O –通过提供回调函数，当 I / O 操作完成并且事件循环调用回调函数时，JavaScript 代码稍后（异步）接收结果，非阻塞 I / O 本质上是异步的，因为当在后台执行 I / O 时，它不会阻止 V8 引擎运行其他 JavaScript 代码。

以往的后端： 开一个端口，在端口用 apache 处理数据

node： 用浏览器的 V8 引擎来运行

node 可以运行 JS 的语法，node 有内置的核心模块：
http 服务，fs 文件操作，url 路径，past 路径处理，os 操作系统等,都在 nodejs 的源代码里写好了可以在源码哪里看到

写 node 的时候要遵循 commonJS 的规范（commonJS 是一个书写 js 的规范）

模块导出: module.exports.add = add

引入: const a = require('add')

内置模块

  • os 模块,获取操作系统的一些信息

  • path 模块,路径处理模块,就是用来处理一些路径

  • url 模块,可以处理网址里面改的参数,截取之类的
  • fs 模块,文件处理,读文件写文件,获取文件名
  • http 模块,比较重要,用来创建请求,发送请求
  • event 模块,事件处理
  • 服务器相应 JSON
  • 流和管道
  • 服务器用流相应 HTM 文件
  • 服务器响应路由,可以吧中间这些判断抽出来变成一个模块方法
  • 使用 post,get 方法获取数据
